Creating a Repository

Definition: A repository is a storage location where your project files and their version history are kept.

Example: To create a repository, navigate to GitHub and click on 'New Repository'. Follow the prompts to initialize it.

Cloning a Repository

Definition: Cloning a repository means making a local copy of a remote repository on your computer.

Example: Use the command git clone [repository URL] to copy a repository to your local machine.

Creating and Managing Branches

Definition: Branching allows you to work on new features or fixes without affecting the main codebase.

Example: Create a new branch using git branch branch-name and switch to it using git checkout branch-name.

Committing and Reverting Commits

Definition: A commit is a snapshot of your code at a specific time, while reverting undoes previous commits.

Example: Use git commit -m "message" to save changes and git revert commit-hash to undo a commit.

Pulling and Pushing Changes

Definition: Pulling updates your local repository with remote changes, while pushing uploads your changes to the remote repository.

Example: Use git pull origin main to fetch changes and git push origin branch-name to upload changes.

Merging and Renaming Branches

Definition: Merging combines changes from different branches, while renaming changes a branch's name.

Example: Merge branches using git merge branch-name and rename with git branch -m old-name new-name.
